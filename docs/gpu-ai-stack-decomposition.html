L0 Blackwell B200 — Die & Package

The B200 is a dual-die package: two reticle-limit dies (TSMC 4NP, ~104B transistors each, 208B total) fused by the NV-HBI die-to-die interconnect at 10 TB/s, so software sees one coherent GPU. Eight HBM3e stacks surround the dies for 192 GB at 8 TB/s.

L0.1 Inside a Streaming Multiprocessor (SM)

The SM is the GPU's core execution engine. Each SM runs thousands of threads grouped in warps of 32, scheduled across 4 processing blocks. Blackwell pairs the classic CUDA-core datapath with 5th-gen Tensor Cores and a 2nd-gen Transformer Engine that adds FP4/FP6 micro-tensor-scaled formats.

UNIFIED L1 DATA CACHE / SHARED MEMORY (~256 KB, configurable split) · TEX UNITS
TMA — TENSOR MEMORY ACCELERATOR (async bulk tensor copies, global ↔ shared)
TENSOR5th-gen Tensor Cores execute matrix-multiply-accumulate on whole warpgroups. New in Blackwell: FP4 & FP6 with micro-tensor scaling (per-32-element scale factors) — the 2nd-gen Transformer Engine picks precision per layer dynamically, doubling throughput and model capacity vs FP8.
EXECExecution model: kernel → grid → thread blocks (CTAs) → warps of 32 threads. GigaThread engine places CTAs on SMs; warp schedulers hide memory latency by switching among resident warps every cycle. Thread-block clusters let CTAs on nearby SMs share distributed shared memory.
DATAData path: TMA streams tiles from HBM → shared memory asynchronously; tensor cores consume tiles from shared memory/registers, so compute and data movement overlap (warp-specialized producer/consumer kernels — the pattern behind FlashAttention-style kernels).
MISCAlso in each SM: special-function units (transcendentals), load/store units, and per-SM instruction caches. FP64 throughput is a small fraction of AI-format throughput — Blackwell is tuned for low-precision AI math.

L3 GB200 NVL72 Rack — one giant GPU

The rack is the new unit of compute: 72 GPUs + 36 Grace CPUs in one NVLink domain, so every GPU can read every other GPU's HBM at full NVLink speed. 130 TB/s of all-to-all fabric bandwidth, ~13.4 TB HBM3e unified, ~1.4 exaFLOPS FP4, ~120 kW, fully liquid-cooled, ~1.36 t.

Each NVLink-5 switch ASIC provides 7.2 TB/s across 72 ports and runs in-network compute (SHARP v4) — reductions like all-reduce execute inside the switches, off-loading the GPUs. Because the whole rack is one NVLink domain, tensor/expert-parallel groups up to 72-way stay off the (much slower) InfiniBand/Ethernet network — the key to trillion-parameter inference throughput.

L4 Cluster / Pod — scale-out fabric

Beyond one rack, GPUs talk over the scale-out network: RDMA fabrics in a rail-optimized fat-tree. A DGX SuperPOD building block is 8× NVL72 racks = 576 GPUs; hyperscale clusters tile this to 100k+ GPUs.
